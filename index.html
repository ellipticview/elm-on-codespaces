1. Introduction

If you want to develop with Elm on Codespaces, this document provides the getting-started instructions.

2. Create a Repository

  1. Log into Github and press the + in the top-right corner. Select new repository.

  2. Provide a name, add a README file and select the .gitignore for Elm

  3. Press [Create Repository]

We now have a repository with this .gitignore file:

# elm-package generated files
elm-stuff
# elm-repl generated files
repl-temp-*

3. Configure the Devcontainer

We will configure the container to include:

  • Elm

  • Elm vscode extension

  • Supporting programs elm-test, elm-format and elm-review

The steps to follow:

  1. Press the green [Code] button.

  2. Select the Codespaces tab

  3. Press the [ …​ ] on the right (Codespace repository configuration)

  4. Select Configure dev container. This will create a default .devcontainer.json.

  5. Copy and paste the snippet below. Your file should then look like this:

    /.devcontainer/devcontainer.json
    {
  "image": "mcr.microsoft.com/devcontainers/universal:2",
  "features": {
    "ghcr.io/devcontainers-contrib/features/elm-asdf:2": {}
  },
  "postCreateCommand": "npm install -g elm-test elm-format elm-review",
  "customizations": {
    "vscode": {
      "extensions": [
        "elmTooling.elm-ls-vscode"
      ]
    }
  }
}

  6. Press the [Commit Changes…​] button and commit directly to the main branch.

4. Create a Codespace

  1. On the code tab, press the green [Code] button

  2. Press [Create codespace on main]. A new browser tab is opened and displays Setting up your codespace. After a while you see the explorer.

5. Create demo program

  1. At the terminal, enter elm init.

  2. In the left menu, click on the Explorer button.

  3. Navigate to the src folder and right-click New File …​. Enter filename Main.elm. This opens the editor.

  4. Paste the sample code from https://elm-lang.org/examples/buttons and save the file.

6. Run the demo program

  1. In the terminal, enter elm reactor

  2. In the bottom-right of your screen, this box appears:

    Press the [Open in Browser] button. The Elm reactor screen appears

  3. Click on the src folder and then on Main.elm

Your program is running!

7. Stop the codespace

  1. Press F1 (or hamburger menu > View > Command palette)

  2. Enter co st to select command Codespaces: Stop current codespace

8. Appendix A: Defining the container step-by-step

Earlier in this document we specified the .devcontainer.json file in one step. For those that would to build it step-by-step, here are the instructions.

8.1. Define the base container

  1. Press the green [Code] button.

  2. Select the Codespaces tab

  3. Press the [ …​ ] on the right (Codespace repository configuration)

  4. Select Configure dev container. This will create a default .devcontainer.json.

  5. In the Marketplace list on the right, enter elm in the search-box and choose Elm (via asdf).

  6. Copy and paste the snippet as instructed. Your file should then look like this:

    /.devcontainer/devcontainer.json
    {
  "image": "mcr.microsoft.com/devcontainers/universal:2",
  "features": {
    "ghcr.io/devcontainers-contrib/features/elm-asdf:2": {}
  }
}

  7. We’ll want a few more programs installed in our codespace. Add a postCreateCommand (remember to add the comma on the preceding line!). The file now looks like this:

    /.devcontainer/devcontainer.json
    {
  "image": "mcr.microsoft.com/devcontainers/universal:2",
  "features": {
    "ghcr.io/devcontainers-contrib/features/elm-asdf:2": {}
  },
  "postCreateCommand": "npm install -g elm-test elm-format elm-review"
}

  8. Press the [Commit Changes…​] button and commit directly to the main branch.

8.2. Create a Codespace

  1. On the code tab, press the green [Code] button

  2. Press [Create codespace on main]. A new browser tab is opened and displays Setting up your codespace. After a while you see the explorer.

8.3. Install VS-Code Elm Extension

  1. Press the Extensions button on the left (button with 4 squares)

  2. Enter elm in the searchbox. Select Elm from Elm tooling and then select [Install in Codespaces].

  3. Select the extension settings and choose add to devcontainer.json
